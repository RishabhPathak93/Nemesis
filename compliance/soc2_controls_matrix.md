# SOC 2 Type II Controls Matrix — CortexView

This matrix maps the SOC 2 Trust Services Criteria to specific CortexView
controls + the evidence sources auditors can pull. Generated by the
Compliance Evidence Collector (`queues/complianceEvidenceQueue.ts`) on
the cadence configured under **Compliance → Evidence**.

| Control | CortexView mechanism | Evidence source |

| CC1.4 Reporting | Scheduled reports + branded HTML/PDF export | `ScheduledReportRun`, `report.export.html` audit |

## Generating an evidence bundle

Operators trigger a snapshot from **Compliance → Evidence**. The bundle
is a tarball under `CV_COMPLIANCE_DIR` containing:

- `access-review.json` — every (User × Role × Membership × isActive) row
- `change-log.json` — git log + audit-log diff for the period
- `uptime.json` — daily aggregates from `/health/deep` history
- `audit-events.json` — full `AuditLog` rows for the period

Auditors typically request a 90-day window. The bundle is regenerable
on demand and the operator hands it over via SFTP / email / VDR.
