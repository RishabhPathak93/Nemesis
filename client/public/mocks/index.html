Report style mocks

Three reporting-layout directions inspired by Faraday's vulnerability-management UI, adapted to Nemesis AI's AI agent context.

Mock 01
Findings table

Dense, scannable list of every finding across all runs. Severity-colored left bar, status pill, owner, asset, CVE-style id. Designed for an analyst working through a backlog.
Mock 02
Finding detail (split view)

A vuln-list on the left, slide-in detail on the right. Description, evidence transcript, mitigations, references, status workflow — all in one view without leaving context.
Mock 03
Executive overview

Manager-facing dashboard: risk score, severity donut, top-5 critical findings, trend over time, compliance coverage. Designed for the engagement summary that lands in a stakeholder's inbox.
Notes:
· All mocks use the same seeded demo data (Demo: Acme Support Bot, etc.) for continuity.
· These are static .html visual prototypes — no React, no API calls. Once a direction is picked we wire it into ReportView.tsx.
· Severity colours and status pills mirror Faraday's convention: critical=red, high=orange, medium=amber, low=blue, info=slate.
