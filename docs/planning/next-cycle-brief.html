2 · Selected next-cycle items — scope + acceptance criteria

Goal: 100% of these leave today with testable acceptance criteria. Each is one independently-shippable PR.

CI-1 + CI-2 — CI pipeline + lint (Ops · M)
In: ci.yml running tsc + vitest + lint (server) and tsc-b + lint + build (client) on PRs; ESLint + lint scripts; remote + branch-protect main. Out: deploy-on-merge, coverage gates, mass reformat (don't touch engine / W2 golden snapshots).
Acceptance: ☐ PR triggers CI; green check required to merge · ☐ npm run lint passes both packages (max-warnings 0) · ☐ 119 server tests + client build run; a broken PR goes red · ☐ origin exists, main protected.
SP-1 — Scan Profiles (Quick / Standard / Thorough) (UX · M)
In: run-tests accepts profile; pure profileToConfig() (tier thresholds + chain depth) threaded per-run; UI selector + est. count; profile stored on the run + shown on the report. Out: per-category selection; changing the default (default = standard == today).
Acceptance: ☐ quick ≈1 case/probe, thorough largest; unit test on profileToConfig() · ☐ default byte-identical to today (back-compat test) · ☐ UI shows profile + est. count pre-run; profile appears on the report.
EV-1 — Evaluator calibration (Accuracy · M · depends on W2 golden — done)
In: EVAL_PROMPT_VERSION flag (default = current); a v2 prompt tuned for fewer false-positives; bench:eval reporting P/R/F1 v1 vs v2 + a ship decision. Out: rewriting the verdict engine; expanding the labeled set beyond ~50.
Acceptance: ☐ bench:eval prints v1 vs v2 macro-P/R/F1 + a recorded decision in bench.json · ☐ v2 improves ≥1 of {P,R,F1} by ≥5% with none regressing below baseline, OR is explicitly not shipped · ☐ default unchanged unless flag set; W2 golden green/intentionally refreshed.
SEC-1 — H-05 refresh token → HttpOnly cookie (Security · M)
In: server sets cv_rt HttpOnly/Secure/SameSite=Strict on login/verify/refresh; /auth/refresh reads cookie-or-body; logout clears it; client stops persisting cv_refresh. Out: moving the access token out of localStorage (follow-up); SSO cookie redesign.
Acceptance: ☐ after login the refresh token is NOT in localStorage; cv_rt HttpOnly cookie set · ☐ silent refresh after access-token expiry works via cookie; logout invalidates · ☐ verified locally end-to-end before any prod deploy; tests green.
TR-1 — Run history + posture trend (UX · M)
In: per-agent "History" list (date, profile, risk score, fails) + risk-over-time sparkline + Δ vs previous run (introduced / resolved by externalId). Out: cross-agent org rollups; CSV export.
Acceptance: ☐ agent detail shows ≥2 prior runs with a risk trend · ☐ selecting two runs shows "introduced N / resolved M" by externalId · ☐ no new DB tables.
LAB-1 ⭐ (stretch) — Dockerize DVRAG + 2 vuln classes (Demo · S)
Acceptance: ☐ docker compose --profile lab up starts DVRAG on :4001 · ☐ two new vuln classes (tool-call SSRF, upload-borne indirect injection) leak under attack · ☐ README updated.
